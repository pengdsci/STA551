ASSIGNMENT #1: Using SQL to create an analytical data by merging the four relational data tables related to pet care data. The data sets are avaiable at https://pengdsci.github.io/datasets/#pet-care.
  • General Requirement: create a single data set that contains the information of pets and their corresponding owners, procedure history and procedure detail. You can use either PROC SQL in SAS or R to run SQL clauses.
  • Specific requirements:
    • Writing a program header file to summary the coding tasks
    • Using relevant SQL clauses to merge different data sets with appropriate keys.
    • Defining a composite key in procedure history and procedure detail and then merge them with other tables
    • Justifying whether the final combined data set is contains the information in all individual data tables
  • Coding Conventions and Formatting:
    • Program header with detailed informaton about author, date, major tasks, etc
    • Code must be well commented.
    • Code must follow the best practices and conventions
  • Submission Requirements:
    • The submitted code should be error-free (tip: run the entire code before submitting it to the D2L drop box).

ASSIGNMENT #2 Choose a data set the meets all requirements specified in the Project Guideline. This data set will be used for the term project focusing on predictive modeling. All subsequent assignments will be the bulding-blocks of the final report of the term project.
This assignment focuses on EDA. The written report should include a meaningful title, an introduction section dewscribing the data set, a section of analysis (EDA) and actions taken based on the EDA. To be more specific, you report should have
  • A meaningful title of the analysis
  • Introduction
    • What will be the question you try to address using this data set?
    • Description of the data source.
    • Sample size and number of feature variables and their types.
    • Information of missing values (the nature, percentage of records with missing values)
    • Methods for handling missing values and justifications.
  • Exploratory Data Analysis (could open several subsections to orgnize your analysis)
    • General objectives of EDA
    • EDA for numeric variables and patterns observed
    • EDA for categorical variables and patterns observed
    • Correlations between numerical variables
    • Potential of dependency between categorical variables
    • Treatments of variables based on EDA results (need justifications)
    • Feature engineering using the above EDA results. Please keep in mind that your engineered data set should have the variables to be used for the subsequent modeling. See the class under the title of EDA for Feature Engineering.
